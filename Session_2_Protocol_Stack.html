Header AnatomyEthernet · L2 Frame
17 / 23
Layer 2 · Data Link

The Ethernet frame — 14 bytes of header, then the world.

The simplest of the headers — but every byte on a typical LAN is wrapped in one. Hover any field.

Destination MAC48 bits
Hardware address of the next-hop device on this link. Switches forward by this field.
Source MAC48 bits
Hardware address of the sending NIC. Used by switches to learn the MAC table.
EtherType16 bits
Tells the receiver what the payload is. 0x0800 = IPv4, 0x0806 = ARP, 0x86DD = IPv6.
Payload46 – 1500 B
The IP packet (or ARP, or other) carried by this frame.
FCS (trailer)32 bits
CRC checksum appended after the payload. Detects corruption on the wire; corrupted frames are silently dropped.
Total frame: ~64 to 1518 bytes on the wire (without VLAN tag) — header + payload + FCS.
Addressing Demultiplex (what's inside?) Payload

Read it as: "I am sending from aa:bb:cc:11:22:33 to aa:bb:cc:44:55:66, and what comes next is an IPv4 packet."

Header AnatomyIPv4 · L3 Packet
18 / 23
Layer 3 · Network

The IPv4 header — 20 bytes that route every packet.

Five rows of 32 bits each. Routers read this to decide where to send the packet next.

0 8 16 24 — 31
Ver4
IP version: 4 for IPv4, 6 for IPv6.
IHL4
Header Length in 32-bit words. Usually 5 (= 20 bytes).
DSCP/ECN8
QoS markings (DSCP) + explicit congestion notification (ECN).
Total Length16
Entire packet length in bytes, header + payload. Max 65,535.
Identification16
Used to reassemble fragments — all fragments of one packet share this ID.
Flags3
Fragmentation control: DF (Don't Fragment), MF (More Fragments).
Fragment Offset13
Position of this fragment in the original packet, in 8-byte units.
TTL8
Time to Live. Each router decrements by 1; at 0 the packet is dropped & ICMP "Time Exceeded" is sent — this is how traceroute works.
Protocol8
What's inside this packet. 6 = TCP, 17 = UDP, 1 = ICMP.
Header Checksum16
Covers the IPv4 header only. Recomputed by every router (because TTL changes).
Source IP Address32
IPv4 of the sender. e.g. 192.168.1.10
Destination IP Address32
IPv4 of the intended receiver. This is what routers look at to forward the packet across the internet.
Routing identity Delivery control Misc / metadata

Header AnatomyTCP · L4 Segment
19 / 23
Layer 4 · Transport

The TCP header — how reliability is made of bytes.

Ports, sequence numbers, and the SYN / ACK / FIN / RST flags every analyst learns to recognize on sight.

0 8 16 24 — 31
Source Port16
Sender's port. Usually an ephemeral high port chosen by the OS (e.g. 49152+).
Destination Port16
Well-known service port: 80 HTTP, 443 HTTPS, 22 SSH, 53 DNS.
Sequence Number32
Position of the first byte in this segment within the overall byte stream. The mechanism that makes TCP ordered.
Acknowledgement Number32
Next byte the sender expects to receive. The mechanism that makes TCP reliable.
Data Off.4
Header length in 32-bit words (because TCP options are variable).
Reserved3
Reserved / future use.
Flags · URG ACK PSH RST SYN FIN9
SYN=open, ACK=acknowledge, FIN=close cleanly, RST=close hard, PSH=deliver now. The three-way handshake = SYN → SYN/ACK → ACK.
Window Size16
How many more bytes the receiver is willing to buffer right now. Drives flow control.
Checksum16
Covers TCP header + payload + a pseudo-header from IP. Detects corruption end-to-end.
Urgent Pointer16
Valid only if URG is set — rarely used in modern apps.
Options + Paddingvariable (0 – 40 B)
Maximum Segment Size (MSS), Window Scale, SACK permitted, Timestamps, … Most performance tuning lives here.
Endpoints (ports) Sequencing & flow control Connection state (flags) Structure / options

Header AnatomyUDP · L4 Datagram
Layer 4 · Transport

The UDP header — 8 bytes, and not one byte more.

Where TCP spends 20+ bytes earning reliability, UDP spends 8 and trusts the application to cope. Two ports, a length, a checksum — done.

0 8 16 24 — 31
Source Port16
Sender's port — often ephemeral, and on a pure one-way stream it may even be 0 (unused).
Destination Port16
Where to deliver: 53 DNS, 67/68 DHCP, 123 NTP, 443 QUIC.
Length16
Length of header + data in bytes. Minimum 8 (header only).
Checksum16
Optional in IPv4, mandatory in IPv6. Covers header, data, and an IP pseudo-header. No retransmit if it fails — the datagram is just dropped.
Data (payload)variable
The application's bytes — a DNS query, a video frame, a game state update. Handed up the instant it arrives.
Endpoints (ports) Length Integrity Payload

Read it as: "Here is a datagram for port 53. Here is how long it is. Take it or leave it — I won't ask again."

LectureTransport choice · TCP vs UDP
Two philosophies of delivery

Reliable conversation, or fast fire-and-forget?

Connection-oriented

TCP

Sets up a connection (3-way handshake), numbers every byte, acknowledges receipt, retransmits losses, and controls flow. You pay in latency and overhead; you get a clean, ordered byte stream.

HTTP/1-2HTTPSSSHSMTPFTP
Connectionless

UDP

No setup, no acknowledgements, no ordering — just address it and send. If a datagram is lost, it's gone. Perfect when "late" is worse than "missing" and the app handles its own recovery.

DNSDHCPVoIPVideoQUIC / HTTP3

Connection Handshake first (SYN → SYN/ACK → ACK) None — send immediately

Ordering Sequence numbers reorder on arrival No ordering guarantee
Header size 20 – 60 bytes 8 bytes
Best when Correctness matters more than speed Speed matters more than completeness

Well-known · registered · ephemeral

A socket = the full address

Network portion · 24 bits

Shared by every device on the LAN — 192.168.1.0. Routers forward by this part; it answers "which network?"

Host portion · 8 bits

Unique to each device — .10. With 8 host bits there are 2⁸ − 2 = 254 usable hosts (minus network & broadcast). It answers "which device?"

IP DesignSubnet masks & CIDR notation
Drawing the boundary

CIDR just counts the network bits.

The mask is a run of 1s

A subnet mask is 32 bits: all 1s for the network, then all 0s for the host. /24 simply means "the first 24 bits are network."

Why classful addressing died

Classes A/B/C locked you to /8, /16, /24 — wildly wasteful. CIDR lets the boundary fall on any bit, so subnets fit the actual host count.

Worked example — 192.168.1.0/26: network .0, broadcast .63, usable hosts .1 – .62, gateway usually .1.

MethodHow to subnet · step by step
A repeatable recipe

Subnetting in four moves.

Count the need. How many subnets, and how many hosts in the largest one? Size for the biggest.
Pick the prefix. Find the smallest mask whose usable hosts cover the need: usable = 2^(host bits) − 2.
Walk the blocks. Subnets begin at multiples of the block size 256 / 2ⁿ in the affected octet — e.g. /26 → .0, .64, .128, .192.
Name the edges. Network = all host bits 0, broadcast = all host bits 1, usable = everything between, gateway usually the first usable.
Quick formula · /(24 + n)

Subnet size = 256 / 2ⁿ addresses
Usable hosts = size − 2
Subnets start at multiples of the size

Worked · /26 (n = 2)

size = 256 / 4 = 64 · usable 62 · subnets at .0 / .64 / .128 / .192.

Lab AWireshark · TCP 3-way handshake · 20 min
Hands-on · 20 minutes

Lab A — catch the handshake.

Capture while you load a website in the browser, then stop after it finishes.
Filter tcp.flags.syn == 1 — every connection setup jumps to the top.
Find the trio: [SYN] → [SYN, ACK] → [ACK] between the same two endpoints.
Right-click → Follow → TCP Stream to read the whole conversation reassembled in order.
Compare a TCP exchange (HTTP) with a UDP one — filter udp.port == 53 and notice: no handshake, just query and reply.
What to look for

Sequence numbers line up

The server's ACK equals the client's SEQ + 1. Wireshark shows relative numbers (starting at 0) by default — that's the same handshake from the previous slide, live.

Tip

Turn on Statistics → Conversations → TCP to see every stream, its packet count, and its duration at a glance.

Lab BPacket Tracer · Simulation Mode walk · 20 min
Hands-on · 20 minutes

Lab B — step the packet by hand.

Build 2 PCs → switch → router → server. Put the PCs and server on different subnets, router as gateway.
Switch to Simulation Mode and ping from a PC to the server.
Watch ARP first: the PC resolves the gateway's MAC before any IP packet can leave the LAN.
Open each PDU — see ICMP encapsulated inside IP inside Ethernet, and the In/Out Layers at every hop.
Confirm the TTL drops by 1 each time the packet crosses the router.
The lesson

Headers in motion

Every field from the anatomy slides — MACs, IP addresses, the Protocol byte, TTL — is right there in the PDU window, changing hop by hop.

Why two subnets?

Same subnet = direct delivery (ARP for the host). Different subnet = the PC ARPs for the gateway and lets the router forward. Watching both makes the rule stick.

Three things
to take home.

01

Headers are readable.

Borrow host bits, halve the hosts, name the edges. The mask is just where you drew the line.
