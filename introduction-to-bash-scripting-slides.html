Chapter 1 · The Basics

What is Bash?

Bourne-Again SHell

Bash is a Unix shell and command language, widely available across operating systems and the default interpreter on most Linux systems.

Use it two ways: interactively in your terminal, or as a full programming language to write scripts.

You'll Learn

Variables · User Input · Comments · Arguments · Arrays · Conditional Expressions · Conditionals · Loops · Functions · Debugging & Testing.

All you need is a UNIX terminal and a text editor — VS Code, Sublime, vim or nano.

From 5× manual setup → 1 shared script

Chapter 2 · Structure

Every script starts with a Shebang

devdojo.sh
# Create the file
touch devdojo.sh   # or: nano devdojo.sh

# The first line MUST point to bash
#!/bin/bash

# Portable alternative — searches $PATH
#!/usr/bin/env bash
.sh extension

Name scripts with a .sh suffix, e.g. devdojo.sh.

What the shebang does

Tells the OS to run the file with the /bin/bash executable.

When to use env

Bash isn't always in /bin (non-Linux / optional installs). /usr/bin/env bash finds it via $PATH.

devdojo.sh

Chapter 4 · Variables

No data types — just name=value

variables
#!/bin/bash

name="DevDojo"          # assign with =
greeting="Hello"

echo "$greeting $name"   # access with $
# → Hello DevDojo
warning No spaces

No spaces around =. Write name="DevDojo", never name = "DevDojo".

Variables hold anything

Numbers or characters — a Bash variable has no declared type.

No semicolons needed

Line breaks end statements; trailing ; is optional (like JS).

Chapter 4 · The #1 Habit

Always quote your variables

quoting
filename="my file.txt"

# WRONG → makes TWO files: "my" and "file.txt"
touch $filename

# RIGHT → one file: "my file.txt"
touch "$filename"
curly braces
greeting="Hello"

echo "$greetings world"    # empty!
echo "${greeting}s world"   # Hellos world
Why quote?

Unquoted values undergo word splitting & globbing — a real source of bugs and security holes.

Rule of thumb

Use "$name", not $name. (Inside [[ ]] splitting won't happen, but quote anyway.)

Braces required for…

Arrays ${a[0]}, slicing ${s:0:3}, defaults ${v:-x}, disambiguation.

Chapter 4 · Command-line input

Reading parameters: $1 $2 $@

./devdojo.sh Bobby buddy!
#!/bin/bash

echo "Hello there $1"   # first param
echo "Hello there $2"   # second param
echo "Hello there $@"   # all params

# Hello there Bobby
# Hello there buddy!
# Hello there Bobby buddy!
Positional variables

Arguments after the script name map to $1, $2, $3 … in the order you pass them.

$@ = everything

A handy way to read all inputs at once, regardless of how many were passed.

Chapter 5 · User Input

Ask the user with read

read
#!/bin/bash

echo "What is your name?"
read name

echo "Hi there $name"
echo "Welcome to DevDojo!"
read -p (shorter)
#!/bin/bash

read -p "What is your name? " name

echo "Hi there $name"
echo "Welcome to DevDojo!"

read prompts the user and stores their typed text in a variable. The -p flag prints a prompt message inline — one fewer echo.

Chapter 6 · Comments

Leave notes with #

commented script
#!/bin/bash

# Ask the user for their name
read -p "What is your name? " name

# Greet the user
echo "Hi there $name"
echo "Welcome to DevDojo!"
# at line start

Anything after # is ignored by Bash and never printed to the screen.

Why comment?

Describe complex logic so teammates (and future-you) can navigate the script with ease.

Chapter 7 · Arguments

Arguments, $@, and $0

./arguments.sh dog cat bird
echo "Argument one is $1"     # dog
echo "Argument two is $2"     # cat
echo "Argument three is $3"   # bird
echo "All arguments: $@"      # dog cat bird

# $0 is the script itself
echo "This file is $0"
rm -f "$0"   # warning self-delete — back up first!
$1 … $n

Each argument after the script name, referenced by position.

$@

All arguments together.

$0

The script's own name — useful for self-reference or self-destruct.

Chapter 8 · Arrays

Many values under one name

arrays
my_array=("value 1" "value 2" "value 3" "value 4")

echo "${my_array[1]}"    # value 2  (0-indexed)
echo "${my_array[-1]}"   # value 4  (last)
echo "${my_array[@]}"    # all elements
echo "${#my_array[@]}"   # 4  (count, with #)
Initialise

Space-separated values inside ( ). Quote each element to keep spaces intact.

warning Braces + quotes

Always use curly braces and double quotes around array expansions.

[@] = all

Prefix with # for the element count.

Chapter 8 · Slicing

Array & string slicing

array slicing (Bash 4+)
array=("A" "B" "C" "D" "E")

echo "${array[@]:1:3}"  # B C D
echo "${array[@]:3}"    # D E  (to end)
${string:start:length}
text="Hello, World!"

echo "${text:7:5}"   # World
echo "${text:7:10}"  # World!  (stops at end)
echo "${text:3}"     # lo, World!

The second number is the maximum length, not an end index (unlike Python). Ask for more characters than remain and Bash simply stops at the end — no error.

Chapter 9 · Conditional Expressions

String, arithmetic & exit tests

text_fields String

-z ${s} → empty
-n ${s} → non-empty
-v name → set
${a} == "${b}"
${a} != "${b}"
${a} < ${b} · ${a} > ${b}

tag Arithmetic

-eq equal
-ne not equal
-lt less than
-le less / equal
-gt greater than
-ge greater / equal

bolt Combine & Exit

[[ a ]] && [[ b ]] # AND
[[ a ]] || [[ b ]] # OR

[[ $? -eq 0 ]] # ok
[[ $? -gt 0 ]] # error

warning In [[ ]], the right side of ==/!= is a glob pattern unless quoted. Quote it for an exact string match.

Chapter 10 · Conditionals

if · elif · else

if / elif / else

if [[ $num -gt 0 ]] ; then
    echo "The number is positive"
elif [[ $num -lt 0 ]] ; then
    echo "The number is negative"
else
    echo "The number is 0"
fi
Structure

if [[ test ]]
then
  commands
fi

Guard example

if (( $EUID == 0 )); then
  echo "Don't run as root"
  exit
fi

Multiple conditions

Chain with && (and) / || (or) to test several things at once.

Chapter 10 · Conditionals

The case statement

case
read -p "Enter your car brand: " car

case $car in
  Tesla)
    echo "Factory is in the USA." ;;
  BMW | Mercedes | Audi | Porsche)
    echo "Factory is in Germany." ;;
  Toyota | Mazda | Subaru)
    echo "Factory is in Japan." ;;
  *)
    echo "Unknown car brand" ;;
esac

case $var in — start, then pattern lists.

) ends a pattern; | ORs several patterns.

Each clause ends with ;;  ·  *) is the default.

Close the block with esac ("case" backwards).

Chapter 11 · Loops

for loops

loop an array
users=("devdojo" "bobby" "tony")

for user in "${users[@]}"
do
    echo "$user"
done
loop a number range
for num in {1..10}
do
    echo "$num"
done

Pattern: for var in <list> → do → commands → done. Quote "${arr[@]}" so elements with spaces are handled correctly. Use {1..10} for numeric sequences.

Chapter 11 · Loops

while vs until

while — run WHILE true
counter=1
while [[ $counter -le 10 ]]
do
    echo "$counter"
    ((counter++))
done
until — run UNTIL true
count=1
until [[ $count -gt 10 ]]
do
    echo "$count"
    ((count++))
done

while repeats while the condition stays true. Increment with ((counter++)) to avoid infinite loops.

until is the mirror image — it repeats until the condition becomes true. Great for input validation.

Chapter 11 · Loops

Control flow: break & continue

continue — skip this iteration
for i in 1 2 3 4 5
do
    if [[ $i -eq 2 ]]; then
        echo "skipping 2"
        continue
    fi
    echo "i is $i"
done
break 2 — exit the OUTER loop
for (( a = 1; a < 10; a++ )); do
  echo "outer: $a"
  for (( b = 1; b < 100; b++ )); do
    if [[ $b -gt 5 ]]; then
        break 2
    fi
    echo "inner: $b"
  done
done

continue [n] stops the current iteration and starts the next; n resumes the n-th enclosing loop.

break [n] ends the loop immediately; break 2 exits two levels — the outer loop too.

Chapter 12 · Functions

Reuse code with functions

functions
function hello() {
    echo "Hello $1!"
}

hello DevDojo      # call — NO parentheses
# → Hello DevDojo!
Two forms

Write function name() { } or just name() { } — the keyword is optional (nicer to read).

warning Calling

Invoke with just the name — no parentheses.

Arguments

Functions receive args exactly like scripts: $1, $2, …

Good functions carry a header comment: Description · Globals · Arguments · Outputs · Returns (0 on success, non-zero on error).

Chapter 14 · Aliases

Create custom bash commands

~/.bashrc
# Long command typed all the time…
netstat -plant | grep '80\|443' | grep -v LISTEN | wc -l

# …becomes a short alias:
alias conn="echo 'Total connections:' ; \
  netstat -plant | grep '80\|443' | grep -v LISTEN | wc -l"

# Now just run:
conn
Make it persistent

Add the alias line to ~/.bashrc (or ~/.bash_profile) so it survives logout.

List them all

Run alias with no arguments to see every alias in your shell.

No root needed

Aliases work without sudo — unlike dropping scripts in /usr/bin.

Chapter 15 · Your First Real Script

A server status report

status.sh
#!/bin/bash
server_name=$(hostname)   # $() runs a command

function memory_check() {
    echo "Memory on ${server_name}:"; free -h
}
function cpu_check()    { uptime; }
function tcp_check()    { wc -l < /proc/net/tcp; }
function kernel_check() { uname -r; }

function all_checks() {
    memory_check; cpu_check; tcp_check; kernel_check
}
all_checks
It reports

Memory usage · CPU load · TCP connections · exact kernel version.

Command substitution

$(hostname) runs the command and stores its output in a variable.

Composition

Small single-purpose functions, wrapped by one all_checks orchestrator.

Chapter 16 · Interactive Menu

A colorful multiple-choice menu

menu.sh
green='\e[32m'; clear='\e[0m'      # color vars
ColorGreen(){ echo -ne "${green}${1}${clear}"; }

menu(){
echo -ne "
$(ColorGreen '1)') Memory usage
$(ColorGreen '2)') CPU load
$(ColorGreen '0)') Exit
Choose an option: "
    read a
    case $a in
        1) memory_check ; menu ;;
        2) cpu_check ; menu ;;
        0) exit 0 ;;
        *) echo "Wrong option."; menu ;;
    esac
}
menu
ANSI colors

Wrap text in escape codes like \e[32m … \e[0m via small helper functions for readable menus.

Recursion = loop

Each case calls menu again, so the menu re-appears after every action until the user picks Exit.

Chapter 17 · Remote Execution

Run one script on many servers

one command, every server
while IFS= read -r server; do
    ssh "your_user@${server}" 'bash -s' < ./remote_check.sh
done < servers.txt
description
servers.txt

One server IP per line — the loop reads through them all.

lan
bash -s

Pipes the local script over SSH into the remote shell's stdin — no copying needed.

rocket_launch
No manual logins

No copying to each box, no individual sessions. Scales to many servers & bigger scripts.

Chapter 18 · jq

Parse JSON in the terminal

jq basics
# install: apt / dnf / pacman / brew install jq

curl "https://api...?apiKey=$KEY" | jq        # pretty-print
curl ... | jq '.[0]'          # first element
curl ... | jq '.[].question'   # one key, all items
What is jq?

A lightweight, flexible command-line JSON processor — portable C, zero runtime dependencies.

Pipe & filter

Pipe any JSON into jq to format, index .[0], or extract keys .[].question.

Chapter 18 · jq in a script

Build a quiz from an API

quiz.sh
url='https://quizapi.io/api/v1/questions?apiKey=KEY&limit=10'
output=$(curl "$url" 2>/dev/null)

output=$(echo "$output" | jq '.[0]')            # first question
question=$(echo "$output" | jq '.question')
answer_a=$(echo "$output" | jq '.answers.answer_a')
answer_b=$(echo "$output" | jq '.answers.answer_b')

echo "
Question: ${question}
A) ${answer_a}
B) ${answer_b}"

Store the API response in a variable, then drill into it step by step with jq.

2>/dev/null hides curl's progress meter so only JSON is captured.

Now Bash can talk to almost any REST API — read, extract, reformat.

Chapter 22 · Password Generator

Random passwords from /dev/urandom

password.sh
#!/bin/bash
read -p "How many characters? " pass_length

# Generate 10 candidates of the chosen length
for i in {1..10}; do
  tr -cd '[:alnum:]' < /dev/urandom \
    | fold -w"${pass_length}" \
    | head -n 1
done
Other one-liners

date | md5sum
date | sha256sum
openssl rand -base64 10

Why urandom?

/dev/urandom is the preferred source — intended for cryptographic use. tr keeps only letters & digits.

warning Security

A practice project — for real secrets use a trusted password manager & 2FA.

Chapter 23 · Redirection

Three streams: STDIN · STDOUT · STDERR

STDIN · 0

Manages input in the terminal.

STDOUT · 1

Normal output text.

STDERR · 2

Error output text.

redirecting streams
echo "Hello" > file.txt        # overwrite (truncates)
echo "World" >> file.txt       # append
ls --hello 2> error.txt        # redirect errors
ls --hello 2> /dev/null         # discard errors
./install.sh > out.txt 2>&1      # both → same file
Chapter 23 · Pipes & Heredocs

Chaining commands & the operator cheat-sheet

pipes · heredoc · herestring
ls | grep ".txt"            # pipe output → input

cat << EOF                  # heredoc (multi-line)
Hello World!
How are you?
EOF

wc <<< "a single line to stdin"   # herestring
Operator cheat-sheet

>  save output to a file
>> append output to a file
<  read input from a file
2> redirect error messages
|  output of one → input of next
<< pipe multiple lines cleanly
<<< pipe a single line cleanly

Pipes connect command → command; redirections connect command ↔ file. Build arbitrary chains: ls -l | tail -n +2 | cut -d ' ' -f3 | sort | uniq -c.

Chapter 25 · Troubleshooting

Run as ./script.sh or add dir to $PATH

Need ; then … fi

unexpected end of file

Unclosed if/for/quote — close with fi/done

integer expression expected

Non-numeric value in arithmetic test

bad substitution

redirection denied

Write to a writable path or sudo

Chapter 25 · Debugging Toolkit

debugging toolkit
