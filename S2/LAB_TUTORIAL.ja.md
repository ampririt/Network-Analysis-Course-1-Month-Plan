## スクリーンショットの追加方法

これらのガイドは中央寄せの番号付き図（`図 N`）を使います。各図にはコメントアウトされた `<img>` 行と、追加するファイル名を示す 📸 キャプションがあります。例:

```html
<p align="center">
  <!-- ![SYN filter applied](./img/labA-step3-synfilter.png) -->
  <em>図3 — 📸 img/labA-step3-synfilter.png: 適用された SYN フィルタ。</em>
</p>
```

画像を表示するには: スクリーンショットをキャプション内の**正確なファイル名**で [`S2/img/`](./img/) に保存し、その上の **`![...]` 行のコメントを外し**ます。自動で中央寄せ表示されます。

## セッション2の練習としてこれで十分か？

**簡潔な答え: 中核目標には十分、追加の補強を2つ提案。**

✅ **ラボA〜Bで十分にカバー:**
- 実際のプロトコルヘッダ（Ethernet / IP / TCP / UDP）の読解 — ラボA
- TCP vs UDP の振る舞い（ハンドシェイク vs コネクションレス）— ラボA
- TCP の信頼性 — シーケンス/ACK番号、セグメント化、グラフ上の輻輳制御 — ラボA
- *ゼロからの*サブネット設計、構築、ルーティングの検証 — ラボB
- 最も一般的な2つの実世界の障害（マスク違い、IP競合）— ラボB

🔹 **定着を高める推奨追加**（各 約10〜15分、任意）:
1. **電卓なしのサブネット化ドリル。** ラボは*1つ*の方式を作るだけ。流暢さは反復から来ます。[宿題の `10.0.0.0/16` 演習](./README.ja.md#宿題) を任意でなく必須として扱いましょう。
2. **2つ目のハンドシェイクの変種。** ラボAで、**TCP 切断**（`tcp.flags.fin == 1`）と **HTTPS** セッションもキャプチャし、ペイロードは暗号化されているがハンドシェイクは依然見えることに注目。

🔸 **後のセッションへ意図的に先送り:** VLAN 理論とトランキング → **セッション4**、DHCP/DNS/ARP サービスの詳細 → **セッション3**、ルーティングプロトコル → **セッション4**。

**結論:** ラボA〜Bはセッション2のすべての学習目標を完全に演習します。流暢さのためにサブネット化ドリル（宿題経由）を、深さのために任意の FIN/HTTPS キャプチャを加えれば、S2 の練習は完成です。
