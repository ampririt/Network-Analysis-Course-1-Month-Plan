**学習目標**: スイッチがMACアドレステーブルを使ってどうフレームを転送するかを理解する。VLANを設定してネットワークを分割する。ルータがどうサブネット間でトラフィックを転送するかを学ぶ。キャプチャでルーティングとスイッチングを解析する。

### 講義
- **レイヤ2 vs レイヤ3**: スイッチングが終わりルーティングが始まる境界。コリジョンドメイン vs ブロードキャストドメイン
- **スイッチング**: MACアドレステーブル、MAC学習とエージング、ブロードキャスト vs ユニキャスト vs マルチキャスト、スイッチの転送判断、未知ユニキャストフレームのフラッディング
- **VLAN**: なぜネットワークを分割するのか（セキュリティ、ブロードキャスト抑制、論理的グループ化）、VLANタグ付け（802.1Q）、トランク vs アクセスポート、ネイティブVLAN
- **スパニングツリープロトコル（STP）**: ループ防止、ルートブリッジ選出、ポート状態（ブロッキング／フォワーディング）— 冗長なスイッチ間リンクがなぜネットワークを溶かさないか
- **ルーティング**: ルーティングテーブル、スタティック vs ダイナミックルーティング、デフォルトゲートウェイ、ロンゲストプレフィックスマッチ、アドミニストレイティブディスタンスとメトリック
- **ダイナミックルーティングプロトコル**: ディスタンスベクタ（RIP）vs リンクステート（OSPF）の概観 — ルータがどう自動で経路を学習するか
- **インターVLANルーティング**: Router-on-a-Stick の概念、サブインターフェース、vs レイヤ3スイッチ（SVI）方式
- **サブネット復習**: ネットワーク／ホスト部、サブネットマスク、CIDR表記、利用可能ホスト数とサブネット範囲の計算

**ラボA — Cisco Packet Tracer：「スモールオフィスネットワーク」メガラボ**

> [!IMPORTANT]
> 本コースの中核となるインフララボです。受講者は現実的なオフィスネットワークをゼロから構築します。

PC6台・スイッチ2台・ルータ1台のオフィスを、**3つのVLAN**（Admin / Sales / IT）で構築します。両スイッチにVLANを作成し、**アクセスポート**を割り当て、スイッチ間およびスイッチ-ルータ間リンクを**トランク**化し、**Router-on-a-Stick** のサブインターフェース（`G0/0.10/.20/.30` を `encapsulation dot1Q` で）をVLANごとのゲートウェイとして設定し、**VLANごとのDHCP**を追加します。その後、インターVLANルーティングを実証し、`show vlan brief` / `show interfaces trunk` / `show ip route` で確認します。

**ラボB — Cisco Packet Tracer：ルーティング比較 — スタティック vs RIP vs OSPF**

**1つ**の3ルータトポロジ（R1–R2–R3、各ルータの背後にLAN）を構築し、PC1からPC3へ**3通りの方法**で到達させます — *同じ*ネットワーク上で、各ルーティング方式のトレードオフを実感できます。

1. **スタティック** — 各リモートLANへの `ip route` を全ルータに手入力する。動作はするが、書いた行数の多さと、リンク断時に何も復旧しないことに注目。
2. **RIP** — スタティックルートを消し、`router rip` / `version 2` / `network …` を有効化する。経路が自動で現れる様子を観察。`show ip route` で**ホップカウント**メトリックとAD **120** を確認。
3. **OSPF** — RIPを外し、`router ospf 1` + `network … area 0` を有効化する。**コスト**メトリック（帯域ベース）とAD **110** を比較し、R1–R2リンクを抜いてOSPFがバックアップ経路に**再収束（reconverge）**する様子を観察。

各段階を `show ip route`・`show ip protocols`・PC1からPC3への `traceroute` で確認します — ルーティングテーブルのソースコード（`S` / `R` / `O`）が、どの方式が効いているかを教えてくれます。

### 詳解：VLAN と 802.1Q トランキング

#### アクセスポート vs トランクポート

| | **アクセスポート** | **トランクポート** |
|:---|:---|:---|
| **運ぶVLAN** | ちょうど**1つ**のVLAN | **複数**のVLAN |
| **フレーム** | タグなし | **802.1Qタグ付き**（ネイティブVLANを除く） |
| **接続先** | エンド機器（PC、プリンタ、AP） | 別のスイッチ、またはルータ（Router-on-a-Stick） |
| **設定** | `switchport mode access` + `switchport access vlan N` | `switchport mode trunk` |

#### 802.1Q タグ（4バイト）

イーサネットヘッダの**送信元MAC**の直後に挿入されます:

| フィールド | ビット | 意味 |
|:---|:---:|:---|
| **TPID** | 16 | 常に `0x8100` — 「802.1Qタグが続く」ことを示す目印 |
| **PCP** | 3 | 優先度／QoSクラス（0〜7） |
| **DEI** | 1 | 廃棄適格インジケータ（drop-eligible） |
| **VLAN ID** | 12 | VLAN番号（**1〜4094**） |

> [!NOTE]
> **ネイティブVLAN**（既定はVLAN 1）は、トランク上で**タグなし**で送られる唯一のVLANです。両端が同じ値で合意している必要があります — *ネイティブVLAN不一致*はVLAN間でトラフィックが漏れる原因になります。

#### インターVLANルーティング：2つの方法

| 方式 | 仕組み | 適する場面 |
|:---|:---|:---|
| **Router-on-a-Stick（ROAS）** | ルータへの1本のトランクリンクに、**VLANごとに1つのサブインターフェース**（`g0/0.10`、`.20`…）を `encapsulation dot1Q` でタグ付け | VLANが少数で、既存のルータを使うとき |
| **レイヤ3スイッチ（SVI）** | `vlan N` の **SVI（Switched Virtual Interface）** がゲートウェイIPを持ち、ハードウェアでルーティング | VLANが多数で、高スループットが必要なとき |

### 詳解：スイッチング・ルーティングと便利な `show` コマンド

#### MACテーブル vs ルーティングテーブル

| | **スイッチ** MACアドレステーブル | **ルータ** ルーティングテーブル |
|:---|:---|:---|
| **レイヤ** | 2（MAC） | 3（IP） |
| **対応付け** | MAC → スイッチの**ポート** | 宛先**ネットワーク** → ネクストホップ／インターフェース |
| **学習方法** | 到着フレームの**送信元MAC**を読む | 接続インターフェース、スタティックルート、ルーティングプロトコル |
| **未知エントリ** | 全ポートに**フラッディング** | 破棄する（またはデフォルトルートを使う） |
| **エージング** | 約5分アイドルで | スタティック＝消えない／ダイナミック＝プロトコルのタイマー |

#### スタティック vs ダイナミックルーティング

| | **スタティック** | **ダイナミック**（RIP / OSPF） |
|:---|:---|:---|
| **設定方法** | 手入力（`ip route …`） | ルータが自動で広告・学習 |
| **規模** | 小規模ネットワーク | 中〜大規模 |
| **障害への反応** | しない（手動で修正） | する（再収束） |
| **オーバーヘッド** | なし | ルーティング更新のCPU／帯域 |

#### アドミニストレイティブディスタンス（情報源が食い違ったときの勝者）

| 経路の情報源 | AD |
|:---|:---:|
| 接続インターフェース（Connected） | **0** |
| スタティックルート | **1** |
| OSPF | **110** |
| RIP | **120** |
| 不明／到達不能 | **255** |

> [!TIP]
> **ADが小さいほうが勝ち。** スタティックルート（AD 1）は、*同じ*ネットワークへのOSPF学習経路（AD 110）に勝ちます。**メトリック**は、**同一プロトコル内**でのタイブレークにのみ使われます。

#### 確認コマンド（Packet Tracer / IOS）

| コマンド | 表示内容 |
|:---|:---|
| `show vlan brief` | どのポートがどのVLANに属するか |
| `show interfaces trunk` | トランクリンク、許可VLAN、ネイティブVLAN |
| `show mac address-table` | 学習したMAC → ポートの対応 |
| `show ip route` | ルーティングテーブル（接続／スタティック／ダイナミック） |
| `show ip interface brief` | インターフェース、IP、up/down状態 |
| `show ip arp` | ルータのIP → MACキャッシュ |

> [!NOTE]
> **パケットの一連の流れ（ルータをまたぐ PC → PC の1フレーム）:** **レイヤ3のIP**アドレスは端から端まで変わりませんが、**レイヤ2のMAC**アドレスは**ホップごとに書き換えられ**、IPの**TTLは各ルータで1減ります** — この3つの事実が本セッション全体を貫きます。
